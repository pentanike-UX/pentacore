# QA 체크리스트

> PR 전·릴리즈 전 빠르게 훑는 용도. 항목은 프로젝트 성장에 맞게 추가한다.

## 공통
- [ ] 주요 플로우가 **끊기지 않음** (해피 패스)
- [ ] **로딩** 상태가 보이거나 중복 제출이 막힘
- [ ] **빈 상태** (데이터 없음) 문구·행동이 있음
- [ ] **에러** 시 사용자에게 이유·다음 행동이 있음 (또는 재시도)
- [ ] **disabled** 상태가 시각적으로 구분됨
- [ ] 모바일 너비(예: 375px)에서 레이아웃 깨짐 없음
- [ ] 데스크톱(해당 시)에서 Figma·스펙과 레이아웃 의도가 맞음
- [ ] 터치 타깃·스크롤 영역이 답답하지 않음

## Figma·스펙 (UI)
- [ ] `docs/04_screen-spec.md`·`docs/05_design-rules.md`와 구현이 대응됨 (프레임·컴포넌트명 추적)
- [ ] 미확정 UI는 **TODO/assumption으로 표시**했거나, 스펙·Figma에 맞게 **확정 반영**함
- [ ] PR에 Figma 대비 차이가 짧게 정리됨 (해당 시)

## 접근성·카피
- [ ] 핵심 버튼 라벨이 짧고 명확함
- [ ] 폼 에러가 필드 근처에 표시됨

## 성능·안정
- [ ] 느린 네트워크에서도 UI가 “죽은 것”처럼 보이지 않음 (TODO: 구체 기준)
- [ ] 콘솔 에러 없음 (알려진 경고는 PR에 명시)

## 보안·데이터 (해당 시)
- [ ] 민감 정보가 클라이언트 로그에 남지 않음
- [ ] 권한 없는 화면 접근 시 적절히 차단
